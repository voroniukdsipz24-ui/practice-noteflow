# NoteFlow: Застосунок для редагування Markdown-нотаток

Мінімалістичний застосунок для створення нотаток у форматі Markdown з живим попереднім переглядом, підсвіткою коду, тимчасовими / самознищуваними нотатками, захистом паролем і обміном у реальному часі через унікальні посилання.

---

### Основна ідея
Поєднати простоту локального редактора з можливістю безпечного обміну нотатками через сервер. Застосунок використовує localStorage як швидкий кеш для роботи без затримок, а сервер виступає як єдине джерело істини для синхронізованих нотаток. Проєкт демонструє роботу клієнт-серверної архітектури, обробку станів між браузером і сервером, а також базові підходи до безпеки веб-застосунків.

Серед ключових можливостей:
- одноразові (read-once) нотатки, які самознищуються після перегляду;
- обмеження часу життя нотатки;
- захист паролем;
- обмін через унікальні посилання без створення акаунтів.

## Як працює інтеграція з базою даних

| Дія | Поведінка фронтенду | Бекенд |
| --- | ------------------- | ------ |
| Створення нотатки | Локально створюється заготовка; одразу відправляється POST-запит на `create.php` | Додається запис; сервер повертає **16-символьний hex `dbId`**  |
| Редагування нотатки | Автозбереження (затримка 900 мс) оновлює localStorage **та** відправляє PUT-запит на `create.php` з `dbId` | Запис оновлюється |
| Налаштування доступу (тимчасова / термін / пароль) | Кнопка "Apply & Generate Link" відправляє POST-запит на `create.php` | Оновлюються `read_once`, `expire_at`, `password_hash`  |
| Посилання для доступу | Скопійований URL: `shared.php?id=<dbId>` | `shared.php` редіректить на `index.html?share=<dbId>` |
| Відкриття посилання | `index.html` виявляє `?share=<dbId>` і робить запит до `view.php` | Повертається JSON нотатки; якщо `read_once=1`, запис видаляється після відповіді |
| Видалення нотатки | Локальне видалення + спроба DELETE-запиту до `delete.php` | Запис видаляється |
| Прострочена нотатка | Очищується при кожному запиті до `view.php` + опціональна подія MySQL | Видаляються записи з `expire_at < NOW()` |

## Швидкий старт (локально, без сервера)

Відкрити `index.html` у браузері. Нотатки зберігаються **в localStorage**. 
> Для обміну між пристроями потрібен PHP + MySQL сервер.

---

## Повне налаштування PHP + MySQL

### 1. Створення бази даних

```bash
mysql -u root -p < schema.sql
```

### 2. Налаштування доступу

Відредагуй `db.php`:

```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'noteflow');
define('DB_USER', 'your_db_user');
define('DB_PASS', 'your_db_password');
```

> `ALLOWED_ORIGIN = '*'` дозволяє запити з будь-якого домену, що зручно для розробки, але небезпечно в публічному доступі. У реальному середовищі слід обмежити це значення до конкретного домену застосунку.

### 3. Розгортання файлів

```
/var/www/html/app/
  index.html
  script.js
  shared.php
  create.php
  view.php
  delete.php
  db.php
  style.css
  base.sql
```

### 4. Доступ

`https://yoursite.com/app/index.html`

## API

### `POST /create.php`

Створення нової нотатки (без `id`).

```json
{
  "id":        "optional-existing-db-id",
  "title":     "My Note",
  "content":   "## Hello\n\nMarkdown content here",
  "read_once": false,
  "expire_in": 86400,
  "password":  "optional-plain-text"
}
```

**Відповідь (201 Created):**

```json
{ "ok": true, "id": "a3f9e12b7c04d8e1" }
```

---

### `GET /view.php?id=<dbId>`

Отримання нотатки. Для захищених нотаток повертає

```json
401 {"error":"password_required"}
```

### `POST /view.php`

Отримання нотатки з паролем:

```json
{ "id": "a3f9e12b7c04d8e1", "password": "secret" }
```

**Відповідь:**

```json
{
  "id": "a3f9e12b7c04d8e1",
  "title": "My Note",
  "content": "...",
  "read_once": true,
  "expire_at": "2026-03-20 14:00:00",
  "has_password": false,
  "destroyed": true
}
```

> `destroyed: true` означає, що нотатка була видалена після цього запиту (режим read-once).

---

### `DELETE /delete.php` або `POST /delete.php`

```json
{ "id": "a3f9e12b7c04d8e1" }
```

## Вимоги

* PHP 8.1+ (використовуються `never`, `match`, іменовані аргументи)
* MySQL 5.7+ або MariaDB 10.3+
* Сучасний браузер (Chrome, Firefox, Safari, Edge)

---

## Особливості роботи та обмеження

1. Застосунок використовує localStorage як кеш, тому локальні нотатки можуть тимчасово відрізнятись від стану на сервері. Усі нотатки, що мають `dbId`, перевіряються через API перед відкриттям, оскільки сервер є єдиним джерелом істини.

2. Одноразові нотатки (read-once) видаляються після першого успішного перегляду. Повторний доступ до них неможливий навіть при наявності посилання або пароля.

3. Застосунок не використовує систему акаунтів, тому доступ до нотаток контролюється через унікальні ідентифікатори (`dbId`) та токени редагування (`edit_token`).

4. У разі відсутності зʼєднання із сервером доступ до нотаток, що вже були синхронізовані, обмежується, щоб уникнути відображення застарілих даних.
